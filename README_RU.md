Экспериментальный вариант туннеля brdige-to-freedom.

По сути дела то же самое, что в бранче main, но вместо своего протокола мульиплексирования и коррекции ошибок использует QUIC завернутый в WS.
Адаптер генерит самоподписанный сертификат при старте, хелпер его не проверяет - не нужно, проверка клиента идет по AUTH_TOKEN.

Конфигурируется точно так же, добавляется только параметр

```
quic:
  mtu: 1200   # QUIC packet size (bytes). Keep ≤ 1200 for best compatibility.
```
у адаптера и хелпера.

Не забудьте передеплоить serverless function перед экспериментами и после.

MAUI-клиента нет, потому что реализация QUIC из .NET, как выяснилось, недоступна на мобильных платформах.
